# 経費精算アラートシステム

経費申請データをAIがリアルタイムで分析し、異常な金額・頻度・不正使用が疑われるパターンを自動検知してアラートレポートを生成します。

---

## クイックスタート

```
/expense-alert <経費データのパスまたは指示>
```

**入力例:**

- `/expense-alert ~/Downloads/expenses_2025.csv` - 経費CSVを分析
- `/expense-alert` - 対話的に入力ソースを確認

---

## 処理フロー

### Step 1: 入力データの読み込み

対応する入力形式:

| 入力タイプ         | 形式   | 処理方法             |
| ------------------ | ------ | -------------------- |
| 経費申請CSV        | .csv   | pandas/csvで読み込み |
| Excel経費台帳      | .xlsx  | openpyxlで読み込み   |
| Google Spreadsheet | URL/ID | Sheets APIで取得     |

### Step 2: 異常検知ルール

AIが以下の観点で経費データを分析:

#### 金額異常

- 同一申請者の過去平均から大幅に逸脱（2標準偏差以上）
- 部門・科目別の上限値を超過
- 端数が不自然（例: 交通費が1,000円単位ばかり）

#### 頻度異常

- 同一日に複数のタクシー利用
- 週末・祝日の経費申請が多い
- 同一取引先への異常な頻度

#### パターン異常

- 分割申請の疑い（承認上限直下の金額が連続）
- 同一金額の反復申請
- 期末に集中する経費申請

#### コンプライアンス違反

- 交際費で1人5,000円超（損金不算入の可能性）
- 10万円以上の備品が消耗品費で計上
- 領収書添付なしの高額申請

### Step 3: アラートレポート生成（Google Spreadsheet）

**出力シート構成:**

1. **アラート一覧**: 検知された異常の一覧（重要度: 高/中/低）
   - 申請者名、日付、金額、科目、異常理由、推奨アクション
2. **申請者別サマリー**: 申請者ごとのリスクスコア・傾向分析
3. **統計ダッシュボード**: 部門別・科目別の異常件数・金額推移
4. **詳細分析**: 各アラートの根拠データと過去比較

### Step 4: 書式設定

- アラート重要度に応じた色分け（高=赤, 中=黄, 低=青）
- ヘッダー行: ダークグレー背景、白文字、太字
- 金額列: `¥#,##0` フォーマット
- 条件付き書式: 高アラートを赤ハイライト
- ヘッダー行固定、フィルター設定

---

## 検知ルールのカスタマイズ

ユーザーが以下を指定可能:

- 金額上限の閾値（科目別）
- 除外する申請者・部門
- 追加の検知ルール
- アラート重要度の基準

---

## 必要な認証情報

Google Sheets API の OAuth トークンが必要:

- トークンパス: `~/.config/gog/sheets_token.json`
- クライアント情報: `~/.config/gog/credentials.json`
- スコープ: `https://www.googleapis.com/auth/spreadsheets`
