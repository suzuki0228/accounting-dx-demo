# 領収書スキャン → データ自動連携スキル

紙の領収書・請求書をスキャンするだけで、OCRによるデータ抽出 → 勘定科目推定 → 会計ソフト連携までを自動化します。

---

## クイックスタート

```
/receipt-scan <スキャン画像のパスまたはフォルダ>
```

**入力例:**

- `/receipt-scan ~/Downloads/scanned_receipts/` - フォルダ内の領収書を一括処理
- `/receipt-scan ~/Downloads/receipt_001.pdf` - 単一の領収書PDF
- `/receipt-scan ~/Downloads/a4_scan.png` - A4用紙スキャン（複数領収書貼付）
- `/receipt-scan` - 対話的に入力ソースを確認

---

## 処理フロー

### Step 1: スキャン画像の読み込み

| 入力形式         | 対応 | 処理                 |
| ---------------- | ---- | -------------------- |
| PDF (単ページ)   | O    | ページ画像を抽出     |
| PDF (複数ページ) | O    | ページごとに分割処理 |
| PNG/JPG (単体)   | O    | そのまま解析         |
| PNG/JPG (A4貼付) | O    | 複数領収書を自動分割 |
| TIFF             | O    | 画像変換後に処理     |

### Step 2: OCR → データ抽出

Claude Codeの画像解析で以下を自動抽出:

| 抽出項目        | 精度目安 | フォールバック       |
| --------------- | -------- | -------------------- |
| 店舗名/取引先   | 95%+     | 「【要確認】」フラグ |
| 日付            | 98%+     | ファイル日付を仮設定 |
| 金額（税込）    | 97%+     | 手動入力促進         |
| 税率（8%/10%）  | 90%+     | 10%をデフォルト      |
| 品目・摘要      | 85%+     | 取引先名で推定       |
| インボイスT番号 | 95%+     | 空欄                 |

### Step 3: 勘定科目の自動推定

取引先名・品目から勘定科目を自動マッピング:

| 取引先/品目パターン          | 勘定科目           |
| ---------------------------- | ------------------ |
| コンビニ、スーパー（飲食物） | 会議費/福利厚生費  |
| 文具、コピー用紙             | 事務用品費         |
| タクシー、駐車場             | 旅費交通費         |
| ホテル、宿泊                 | 旅費交通費（宿泊） |
| 飲食店（5,000円超/人）       | 交際費             |
| ガソリン                     | 車両費             |
| 電話、通信                   | 通信費             |

### Step 4: Google Spreadsheet出力

**出力シート構成:**

1. **OCR結果一覧**: 全領収書の抽出データ
   - ファイル名, 取引先, 日付, 金額, 税率, 品目, 勘定科目, 信頼度, T番号
2. **勘定科目別集計**: 科目ごとの件数・金額合計
3. **要確認リスト**: 信頼度が低い or 異常値の項目
4. **freee連携用CSV**: freeeインポート形式のデータ

### Step 5: 書式設定

- ヘッダー行: ネイビー背景、白文字
- 信頼度「高」: 白背景
- 信頼度「中」: 薄黄背景
- 信頼度「低」: 薄赤背景 + 「要確認」フラグ
- 金額列: `¥#,##0` フォーマット

## freee連携

出力CSVをfreeeにインポート可能なフォーマットで生成:

```csv
日付,勘定科目,金額,税区分,取引先,摘要
2026-03-01,旅費交通費,12000,課対仕入10%,JR東日本,出張交通費
2026-03-03,消耗品費,5400,課対仕入10%,Amazon,コピー用紙
```
