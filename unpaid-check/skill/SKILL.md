# 未入金チェックスキル

売掛金台帳と入金データをAIが自動照合し、未入金の取引先を即時特定。未入金リストと督促テンプレートを自動生成します。

---

## クイックスタート

```
/unpaid-check <売掛金データのパスまたは指示>
```

**入力例:**

- `/unpaid-check ~/Downloads/accounts_receivable.csv` - 売掛金CSVを分析
- `/unpaid-check ~/Downloads/ar_data/ ~/Downloads/bank_statement.csv` - 売掛金+入金データ
- `/unpaid-check` - 対話的に入力ソースを確認

---

## 処理フロー

### Step 1: 入力データの読み込み

2種類のデータを突合:

| データ     | 形式            | 必要項目                             |
| ---------- | --------------- | ------------------------------------ |
| 売掛金台帳 | CSV/XLSX/GSheet | 取引先名, 請求日, 請求金額, 入金期日 |
| 入金データ | CSV/XLSX/GSheet | 振込人名, 入金日, 入金金額           |

### Step 2: 自動照合（マッチング）

AIが以下のロジックで突合:

1. **完全一致**: 取引先名+金額が完全に一致
2. **名寄せマッチ**: 取引先名の表記揺れを吸収（株式会社/（株）、全角/半角等）
3. **金額近似マッチ**: 振込手数料差分（-440円/-660円）を考慮
4. **分割入金検知**: 1つの請求に対する複数回入金を検知

### Step 3: 未入金リスト生成（Google Spreadsheet）

**出力シート構成:**

1. **未入金一覧**: 未入金の取引先リスト
   - 取引先名, 請求日, 請求金額, 入金期日, 延滞日数, ステータス（未入金/一部入金/延滞）
2. **照合結果**: 全件の照合結果（入金済み/未入金/不明）
3. **督促テンプレート**: 取引先別の督促メール文面テンプレート
   - 1回目督促（丁寧）、2回目督促（やや強め）、3回目督促（最終通知）
4. **集計サマリー**: 未入金総額、延滞ランク別集計、取引先別未回収額

### Step 4: 書式設定

- 延滞日数に応じた色分け（30日以上=赤, 15日以上=黄, それ以下=緑）
- ヘッダー行: ダークグレー背景、白文字、太字
- 金額列: `¥#,##0` フォーマット
- ヘッダー行固定、フィルター設定

---

## 督促テンプレート

延滞レベルに応じた3段階の督促文面を自動生成:

| レベル | タイミング        | トーン             |
| ------ | ----------------- | ------------------ |
| 1回目  | 入金期日+5営業日  | お知らせ・確認依頼 |
| 2回目  | 入金期日+15営業日 | 再度のご連絡       |
| 3回目  | 入金期日+30営業日 | 最終ご確認のお願い |

---

## 必要な認証情報

Google Sheets API の OAuth トークンが必要:

- トークンパス: `~/.config/gog/sheets_token.json`
- クライアント情報: `~/.config/gog/credentials.json`
- スコープ: `https://www.googleapis.com/auth/spreadsheets`
